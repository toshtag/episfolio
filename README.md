# Episfolio

> Episode + Portfolio — 業務エピソードと人生レイヤーの素材を蓄積し、根拠あるキャリアドキュメント群（履歴書・職務経歴書・プラスアルファ書類・面接準備資料）に変換するローカルファースト・デスクトップアプリ。

## ステータス

🟢 **v0.1.0 リリース済（2026-04-29）** — `pnpm tauri dev` での開発ビルド動作確認済み。署名済の配布バイナリは未配布（v1.0 で対応予定）。

次フェーズは **v0.2（自分大全 + 改訂履歴強化）** に着手中。

## 何をするツールか

転職活動の「ドキュメントポートフォリオ」を継続的に育てるツールです。履歴書・職務経歴書だけでなく、書籍メソッドが推奨する各種ドキュメント（自分大全＝人生年表、面接の赤本、上司/顧客リファレンス、職務経歴ダイジェスト、エージェント連携書類など）を求人ごとに何度も書き換えながら、編集履歴を「いつ・なぜ・誰宛に書いたか」付きで残すことを支援します。

ロジックで対応できるところはテンプレート・チェックリスト・構造化テーブルで解決し、AI は壁打ち・下書き役に閉じます（v0.5 以降）。

### v0.1 で使える機能

- 業務エピソードの構造化保存・編集・削除（17 フィールドの STAR 構造 + 再現性 + 外部評価）
- スキルエビデンスの手書き作成（AI 抽出は v0.5 で復活予定）
- キャリアドキュメント（自己 PR / 履歴書）の手動編集と DocumentRevision 管理
- Markdown / JSON エクスポート
- OS keychain への API key 保存（v0.5 の AI 復活に備えた基盤、v0.1 では UI hide）
- **完全ローカル動作**：v0.1 では外部 LLM への送信機能を一切外す決定（ADR-0007）

## 設計方針

- **ローカルファースト**：データはあなたの端末に留まる。v0.1 は完全ローカル、v0.5 以降の AI 機能でも送信内容は明示的にプレビューしてから送る
- **AI は候補生成役、人間が判断**：AI 出力は常に candidate。採用 / 修正 / 却下を人間が決める。AI を介さない手書きパスを恒久的に維持する
- **データのポータビリティ**：Markdown / JSON でいつでもエクスポート・インポート
- **長期保守性 > 短期開発速度**：流行りの技術より長寿命な選択
- **完全 OSS（MIT License）**

## 技術スタック

| レイヤー | 技術 |
|---|---|
| Desktop shell | Tauri v2 |
| UI | Lit (Web Components) + TypeScript |
| Native boundary | Rust（SQLite / OS keychain / FS / HTTP のみ） |
| Database | SQLite（FTS5 + JSON1） |
| Schema | Zod |
| AI（v0.5 で復活） | OpenAI API + ローカル LLM（Ollama 等の OpenAI 互換 API） |
| Build | Vite + Cargo |
| Lint | Biome |
| Test | Vitest + cargo test |

公開設計ドキュメントは [docs/adr/](./docs/adr/) を参照。
