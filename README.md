# Episfolio

> Episode + Portfolio — 業務エピソードと人生レイヤーの素材を蓄積し、根拠あるキャリアドキュメント群（履歴書・職務経歴書・プラスアルファ書類・面接準備資料）に変換するローカルファースト・デスクトップアプリ。

## ステータス

🟢 **v0.4.0 リリース済（2026-05-02）** — `pnpm tauri dev` での開発ビルド動作確認済み。署名済の配布バイナリは未配布（v1.0 で対応予定）。

v0.4.0 は面接の赤本（QA 集）・面接後報告シート・エージェント実績表・面談メール・求人希望条件シートを追加した機能リリース。詳細は [CHANGELOG.md](./CHANGELOG.md) を参照。

次フェーズは **v0.5（AI 壁打ち・下書き機能の復活）**。

## 何をするツールか

転職活動の「ドキュメントポートフォリオ」を継続的に育てるツールです。履歴書・職務経歴書だけでなく、人生年表（自分大全）、上司・顧客リファレンス、職務経歴ダイジェスト、面接準備資料、エージェント連携書類など、転職活動で必要になる各種ドキュメントを求人ごとに何度も書き換えながら、編集履歴を「いつ・なぜ・誰宛に書いたか」付きで残すことを支援します。

ロジックで対応できるところはテンプレート・チェックリスト・構造化テーブルで解決し、AI は壁打ち・下書き役に閉じます（v0.5 以降）。

### v0.4 で使える機能

- 業務エピソードの構造化保存・編集・削除（17 フィールドの STAR 構造 + 再現性 + 外部評価）
- スキルエビデンスの手書き作成（AI 抽出は v0.5 で復活予定）
- キャリアドキュメント（自己 PR / 履歴書）の手動編集と DocumentRevision 管理（改訂理由・宛先メモ・対象求人・前バージョン参照付き）
- **人生年表（自分大全）** の記録・編集・削除
- **JobTarget（求人連携）**：求人ごとに会社名・職種・必須要件・歓迎要件を構造化保存
- **職務経歴ダイジェスト**：必須要件 × Episode マッピング UI と Markdown プレビュー
- **面接の赤本**：求人単位で面接 Q&A を構造化保存・並び替え
- **面接後報告シート**：面接結果・自己評価・次アクションの記録
- **エージェント実績表**：転職エージェントの情報・評価を一元管理
- **面談メール**：エージェント別に面談メールを記録・Markdown エクスポート
- **求人希望条件シート**：A/B/C グループ別企業リストと希望条件を構造化保存・Markdown プレビュー
- **完全ローカル動作**：外部 LLM への送信機能は v0.5 まで提供しない（ADR-0007 / ADR-0009）。v0.2.1 で AI 送信 / API key 関連の Tauri command と OpenAI adapter を物理削除し、CSP も strict policy 化済

## 設計方針

- **ローカルファースト**：データはあなたの端末に留まる。v0.3 は完全ローカル（外部送信 surface はコードレベルで存在しない）。v0.5 以降の AI 機能でも送信内容は明示的にプレビューしてから送る
- **AI は候補生成役、人間が判断**：AI 出力は常に candidate。採用 / 修正 / 却下を人間が決める。AI を介さない手書きパスを恒久的に維持する
- **データのポータビリティ**：Markdown / JSON / PDF / DOCX でいつでもエクスポート・インポート（v0.7 で実装予定）
- **長期保守性 > 短期開発速度**：流行りの技術より長寿命な選択
- **完全 OSS（MIT License）**

## 技術スタック

| レイヤー | 技術 |
|---|---|
| Desktop shell | Tauri v2 |
| UI | Lit (Web Components) + TypeScript |
| Native boundary | Rust（SQLite / OS keychain / FS / HTTP のみ） |
| Database | SQLite（FTS5 + JSON1） |
| Schema | Zod |
| AI（v0.5 で復活） | OpenAI API + ローカル LLM（Ollama 等の OpenAI 互換 API） |
| Build | Vite + Cargo |
| Lint | Biome |
| Test | Vitest + cargo test |

公開設計ドキュメントは [docs/adr/](./docs/adr/) を参照。
